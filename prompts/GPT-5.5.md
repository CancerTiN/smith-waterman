一、总体目标

二、算法要求

实现 Smith-Waterman 局部序列比对，使用线性 gap 分值。

给定：

* seq1 作为矩阵行方向。
* seq2 作为矩阵列方向。
* 得分矩阵 H 的尺寸为 seq1.length + 1 行、seq2.length + 1 列。
* 第 0 行、第 0 列全部初始化为 0。

对 i >= 1 且 j >= 1 的每个单元格，使用以下递推逻辑：

* diag = H[i - 1][j - 1] + 当前字符匹配或错配得分。
* 如果 seq1[i - 1] 等于 seq2[j - 1]，当前字符得分为 match。
* 如果 seq1[i - 1] 不等于 seq2[j - 1]，当前字符得分为 mismatch。
* up = H[i - 1][j] + gap。
* left = H[i][j - 1] + gap。
* zero = 0。
* H[i][j] = max(zero, diag, up, left)。

默认评分参数：

* match = 2
* mismatch = -1
* gap = -1

重要注意事项：

* gap 输入值本身就是参与相加的分值，默认为 -1，不要再额外取负。
* 不要实现 affine gap，只使用线性 gap。
* 算法核心必须与动画逻辑分离：先完整计算矩阵、pointer 矩阵、填充步骤和回溯步骤，再播放动画。
* computeSmithWaterman 函数只负责算法计算，不直接操作 DOM。

三、traceback 方向矩阵

同时保存 traceback 方向矩阵 pointer。

pointer 取值规则：

* diag：当前单元格来自左上方，表示 seq1[i - 1] 与 seq2[j - 1] 对齐。
* up：当前单元格来自上方，表示 seq1[i - 1] 与 gap 对齐。
* left：当前单元格来自左方，表示 gap 与 seq2[j - 1] 对齐。
* none：当前单元格为 0，回溯终止。

若多个候选方向得分相同，使用确定性优先级：

diag 优先于 up，up 优先于 left，left 优先于 none。

具体规则：

1. 如果最大值等于 diag 且大于 0，pointer 为 diag。
2. 否则如果最大值等于 up 且大于 0，pointer 为 up。
3. 否则如果最大值等于 left 且大于 0，pointer 为 left。
4. 否则 pointer 为 none。

矩阵填充完成后：

* 找出全矩阵最高分单元格。
* 若最高分有多个，选择行优先扫描遇到的第一个最高分单元格。
* 从该最高分单元格开始回溯。
* 回溯直到当前单元格分值为 0 或 pointer 为 none。
* 回溯过程中记录路径和比对字符。
* 因为回溯方向是从右下往左上，最终展示前需要反转构建出的比对数组。

四、回溯构建规则

回溯时按照 pointer 方向构建三行比对结果。

如果方向为 diag：

* alignedSeq1 加入 seq1[i - 1]。
* alignedSeq2 加入 seq2[j - 1]。
* 如果两个字符相同，matchLine 加入竖线。
* 如果两个字符不同，matchLine 加入空格。
* i 减 1，j 减 1。

如果方向为 up：

* alignedSeq1 加入 seq1[i - 1]。
* alignedSeq2 加入短横线。
* matchLine 加入空格。
* i 减 1。

如果方向为 left：

* alignedSeq1 加入短横线。
* alignedSeq2 加入 seq2[j - 1]。
* matchLine 加入空格。
* j 减 1。

如果最高分为 0，则不要回溯，展示提示：

没有找到正分局部比对，请调整序列或评分参数。
