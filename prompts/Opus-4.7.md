## 一、算法规范（必须严格遵守）

### 1.1 递推公式（线性 gap 罚分）
对长度为 m 的序列 A 和长度为 n 的序列 B，构建 (m+1) × (n+1) 的得分矩阵 H：

```
H(i, 0) = H(0, j) = 0
H(i, j) = max {
    0,                              // 局部比对的关键：允许"重新开始"
    H(i-1, j-1) + s(A[i], B[j]),    // 对角：匹配或错配
    H(i-1, j)   + gap,              // 上：A 中字符对应 B 的空位
    H(i, j-1)   + gap               // 左：B 中字符对应 A 的空位
}
其中 s(a,b) = match (若 a==b) 或 mismatch (若 a!=b)
```

### 1.2 回溯指针（必须在填充阶段同时记录）
每个单元格除了存得分外，还要存一个 **traceback pointer**，取值之一：
- `DIAG`（↖，得分来自对角）
- `UP`（↑，来自上方）
- `LEFT`（←，来自左方）
- `STOP`（值为 0，回溯终止点）

回溯阶段**必须沿指针方向走**，而不是临时比较邻居最大值。这是 SW 算法的核心数据结构，请勿简化。

### 1.3 与 Needleman-Wunsch 的关键差异（页面需用 tooltip 或说明文本指出）
- max 中包含 0（允许从任意位置重新开始局部比对）
- 回溯起点是**全矩阵最大值**，不是右下角
- 回溯终点是**值为 0 的单元格**，不是 (0,0)

### 1.4 多最优解
- 若存在多个值相等的最大单元格，取行/列索引字典序最小者作为回溯起点
- 在 UI 上以小标记（如 ★）标出所有最大单元格，提示用户存在并列最优

## 二、输入与控件

### 2.1 序列输入
- 两个文本框，标签 "Sequence A"、"Sequence B"
- 接受 DNA（ACGTN）或蛋白质（20 AA + X）单字母码，**大小写不敏感**，自动转为大写
- **校验规则**：
  - 空输入：禁用"运行"按钮并提示
  - 非法字符：实时红框提示具体非法字符
  - 长度上限：每条序列 ≤ 30，超出时拒绝运行并提示（避免矩阵过大）
- 提供两个示例按钮：
  - "DNA 示例"：A = `GGTTGACTA`, B = `TGTTACGG`（预期最优局部得分 = 13，最优比对为 `GTT-AC` / `GTTGAC`，match=+3, mismatch=-3, gap=-2 时的经典 Wikipedia 示例）
  - "蛋白质示例"：A = `HEAGAWGHEE`, B = `PAWHEAE`（teaching 用，可自由验证）

### 2.2 得分参数
- match（默认 +2，范围 0~10）
- mismatch（默认 -1，范围 -10~0）
- gap（默认 -1，范围 -10~0，线性罚分）
- 不强制符号约束，但鼠标悬停参数标签时显示"通常 match > 0, mismatch ≤ 0, gap ≤ 0"的提示

### 2.3 动画控制
- **运行**按钮（重复点击会先重置再开始）
- **重置**按钮
- **暂停 / 继续**按钮
- **单步前进**按钮（暂停状态下可用，推进一格填充或一步回溯）
- **速度滑块**：5 档（极慢 1500ms / 慢 800ms / 中 400ms / 快 150ms / 极快 50ms 每步）
- **跳过填充动画**勾选框（直接展示填好的矩阵，只演示回溯）

## 三、可视化要求

### 3.1 矩阵填充动画（第一阶段）
- 用 HTML Table 实现（便于无障碍和文本选择），单元格 CSS transition 控制颜色淡入
- 顶部行显示 Sequence B 字符，左侧列显示 Sequence A 字符，含 "−" 占位列/行（对应 i=0 / j=0）
- 按行优先顺序逐格点亮：
  - **当前格**：暖色背景（如 #FFB347）+ 轻微 scale 1.1
  - **三个依赖邻居**：浅蓝高亮（↖、↑、←）
  - **当前行/列对应的序列字符**：加粗 + 暖色下划线
- 矩阵右侧固定一个"计算面板"，显示当前步的代入计算，例如：
```
  H(3, 2) = max {
      0,
      H(2,1) + s(C,C) = 4 + 2 = 6   ✓ ↖
      H(2,2) + gap   = 5 + (-1) = 4
      H(3,1) + gap   = 3 + (-1) = 2
  } = 6
```
  被选中的项（即指针来源）用 ✓ 和方向箭头标注。
- 每个已填充单元格 hover 时显示 tooltip：`H(i,j)=X, ptr=↖/↑/←/STOP`

### 3.2 回溯动画（第二阶段）
- 矩阵填充完成后，**全矩阵最大单元格**用 ★ 标记并闪烁两次
- 沿 traceback pointer 逐格移动：
  - 当前回溯格：红色边框
  - 走过的路径：用半透明红色线条连接单元格中心（可用 SVG overlay 或 CSS 伪元素）
  - 同步在矩阵下方逐字符构建对齐字符串：
```
    A: G G T T - A C
       | | | |   | |
    B: G T T G - A C   (示意，- 表示 gap)
```
- 遇到值为 0 的单元格时停止，闪烁该格表示终止

### 3.3 最终结果面板
回溯结束后，在页面下方展示：
- **比对得分**：大字号
- **比对长度、匹配数、错配数、空位数**统计
- **对齐三行**（A / 匹配标识 / B），等宽字体（如 `ui-monospace`、`Menlo`、`Consolas`）
  - 匹配位置用 `|`
  - 错配位置用 `.` （比空格更可见）
  - 空位位置用 ` `（空格）
- **复制结果**按钮，复制为纯文本：
```
  Score: 13
  Length: 8, Identity: 6/8 (75%), Gaps: 2/8 (25%)
  A: GGTTGAC-T
     ||| ||| 
  B: GGT-GACGT
```

## 五、技术实现要点

- **离散步进**（每格填充、每步回溯）用 `setTimeout`，由速度滑块控制间隔
- **连续过渡**（颜色、缩放、路径绘制）用 CSS `transition` 或 `requestAnimationFrame`
- **不使用 `setInterval`**，避免动画堆积
- **状态机**：定义清晰的状态 `IDLE / FILLING / FILLED / TRACING / DONE`，按钮可用性根据状态切换
- **代码组织**（在单文件内分块、加注释）：
  1. 算法层：`computeMatrix(A, B, params)` 纯函数，返回 `{H, ptr, maxCell}`
  2. 动画控制层：基于状态机，逐步消费算法层产生的事件序列
  3. 渲染层：DOM 操作集中在少数函数中
- **可中断**：重置时清除所有 pending 的 setTimeout（保存 timer id）
